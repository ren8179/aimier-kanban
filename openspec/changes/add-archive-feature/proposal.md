# Proposal: Add Task Archive Feature

## Intent
当前爱弥儿任务看板的"已完成"列会无限增长，导致界面臃肿、加载缓慢。需要实现自动归档功能，当已完成任务超过10条时，自动将最旧的任务归档到单独的存储中，提升性能和用户体验。

## Scope
- 自动归档机制：已完成任务超过10条时，按时间顺序自动归档多余任务
- 归档存储：将归档任务保存到单独的 JSON 文件中
- 懒加载：默认不加载归档内容，减少内存占用
- 查看归档：提供"查看归档"功能，按需加载归档数据
- 统计更新：归档任务不计入默认统计

## Approach
- 后端：新增归档 API 端点，修改任务状态更新逻辑，自动触发归档
- 前端：添加"查看归档"按钮和模态框，实现归档任务的分页显示
- 数据存储：使用 `data/archive/` 目录按月存储归档任务

## Success Criteria
- [x] 已完成任务超过10条时自动归档最旧的任务
- [x] 归档任务从主任务列表中移除
- [x] 页面加载时不加载归档数据
- [x] 用户可以查看所有归档任务
- [x] 归档任务可以从归档中恢复
